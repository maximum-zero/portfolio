# Architecture Decision Records

## 철학
콘텐츠의 신뢰성이 기술 복잡도보다 우선. 정적 생성으로 빠르게, 구조는 단순하게, 나중에 쉽게 확장 가능하게.

---

### ADR-001: Next.js 15 App Router 선택
**결정**: Next.js 15, App Router 방식  
**이유**: Server Component로 불필요한 JS 번들을 줄일 수 있고, `next/og`(Edge Runtime) OG 이미지 생성이 내장되어 별도 설정이 없다. Contact는 단순 링크로 처리하므로 API Route 불필요  
**트레이드오프**: Astro 대비 번들 사이즈가 크지만, 추후 동적 기능 추가 시 확장이 용이하다

---

### ADR-002: 콘텐츠를 CMS 없이 data/ 정적 파일로 관리
**결정**: career.ts, projects.ts 등 TypeScript 파일에 콘텐츠 직접 작성  
**이유**: 포트폴리오 콘텐츠는 업데이트 빈도가 낮다. Contentful/Sanity 등 CMS를 붙이면 외부 의존성과 비용이 생긴다. 개발자가 코드로 관리하는 것이 더 자연스럽다  
**트레이드오프**: 콘텐츠 수정 시 재배포 필요. 단, Vercel 자동 배포이므로 push로 해결됨

---

### ADR-003: 블로그는 외부 링크로 처리
**결정**: 자체 블로그 라우트 없이 Contact/Header에 외부 블로그 URL 링크만 포함  
**이유**: 포트폴리오 사이트에서 블로그 글을 직접 호스팅할 필요가 없다. 외부 블로그 링크로 충분하며, 구조를 단순하게 유지할 수 있다  
**트레이드오프**: 블로그 디자인이 포트폴리오 사이트와 별개로 관리됨. 단, 유지 비용이 없다

---

### ADR-004: 미니멀 파스텔 라이트 모드 단일 테마
**결정**: 다크모드 없이 라이트 모드 단일 고정. 색상은 웜 오프화이트 + 세이지 그린 포인트  
**이유**: 다크/라이트 전환 지원 시 양쪽 모두 완성도 있게 만들어야 한다. 한쪽을 완벽하게 만드는 것이 둘을 반쪽씩 만드는 것보다 낫다. 파스텔 미니멀은 라이트 모드에서 의도가 명확히 전달됨  
**트레이드오프**: 다크모드 선호 방문자 경험 미지원. 추후 `dark:` 클래스 추가는 가능

---

### ADR-005: 프로젝트 상세를 Drawer(슬라이드 패널)로 처리
**결정**: 프로젝트 카드 클릭 시 별도 페이지 이동 없이 우측에서 좌로 슬라이드인되는 Drawer UI 사용  
**이유**: 단일 페이지 흐름을 끊지 않으면서 상세 정보를 충분히 보여줄 수 있다. 뒤로가기 없이 자연스럽게 닫을 수 있어 UX가 부드럽다  
**트레이드오프**: 프로젝트별 URL이 없으므로 직접 링크 공유 불가. 단, 포트폴리오 특성상 공유보다 탐색이 우선

---

### ADR-007: Projects 섹션 카테고리 필터
**결정**: 전체 / 백엔드 / 풀스택 / 프론트엔드 탭 필터 추가  
**이유**: 백엔드·풀스택 양쪽으로 지원하는 상황에서, 방문자(채용 담당자)가 관심 있는 프로젝트만 빠르게 볼 수 있어야 한다. 프로젝트 수가 늘어나도 카드 그리드가 과밀해지지 않음  
**구현**: `Projects.tsx`(서버)는 전체 데이터를 내려주고, `ProjectList.tsx`(클라이언트)가 필터 상태를 `useState`로 관리. 해당 카테고리 프로젝트가 없으면 탭 비활성 처리  
**트레이드오프**: 현재 프로젝트가 2개뿐이어서 초기에는 필터 효용이 적다. 단, 회사 프로젝트 추가 이후 즉시 활용 가능

---

### ADR-006: Framer Motion 제한적 사용
**결정**: Framer Motion 도입하되, fade-in / slide-up / Drawer 슬라이드인 세 가지만 허용. 섹션 진입 애니메이션은 1회만 실행  
**이유**: Drawer의 슬라이드인 트랜지션을 CSS만으로 구현하면 Framer Motion과 중복 의존성이 생긴다. 하나로 통일하는 것이 낫다  
**트레이드오프**: 번들 크기(~25kB gzipped) 추가. `prefers-reduced-motion` 대응 필수
